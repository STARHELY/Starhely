# 巧家青花椒溯源系统 - 部署说明

## 系统架构

```
一个GitHub仓库 → 一个网站 → 多个批次二维码
```

所有批次共用同一个页面（index.html），通过URL参数 `?id=xxx` 区分不同批次。
数据存储在 `products.json` 中，后台管理（admin.html）导出后上传到GitHub即可更新。

## 文件清单

| 文件 | 说明 |
|------|------|
| index.html | 溯源展示页面（顾客扫码看到的） |
| admin.html | 后台管理系统（你用的） |
| products.json | 产品数据库（所有批次信息） |
| img_cover.jpg | 封面图 |
| img_origin_farm.jpg | 产地实拍图 |
| img_farmer.jpg | 农户照片 |
| img_report.jpg | 检测报告图 |
| img_product.jpg | 产品包装图 |

## 部署步骤（一次性）

### 1. 创建GitHub仓库
- 打开 https://github.com/new
- 仓库名：`qiaojia-trace`（或你喜欢的名字）
- 设为 **Public**（GitHub Pages免费版只支持公开仓库）
- 点 Create

### 2. 上传文件
- 在仓库页面点 "uploading an existing file" 或 "Add file → Upload files"
- 把以上所有文件拖进去
- 点 Commit changes

### 3. 开启GitHub Pages
- 进入仓库 Settings → Pages
- Source 选择 "Deploy from a branch"
- Branch 选择 `main`，文件夹选 `/ (root)`
- 点 Save
- 等待1-2分钟

### 4. 获取访问地址
部署成功后，你的溯源系统地址是：
```
https://你的用户名.github.io/qiaojia-trace/
```

溯源页面：`https://你的用户名.github.io/qiaojia-trace/index.html?id=QJ2024052001`
管理后台：`https://你的用户名.github.io/qiaojia-trace/admin.html`

## 日常使用流程

### 添加新批次
1. 打开 admin.html（本地或GitHub Pages都行）
2. 点「新增批次」→ 填写农户、日期等信息
3. 点「保存」
4. 点「📤 导出JSON」→ 下载 products.json
5. 去GitHub仓库，点 products.json → 编辑（铅笔图标）→ 用新文件内容替换 → Commit
6. 等待1分钟生效

### 生成新批次的二维码
1. 在 admin.html 的产品列表中，找到对应批次
2. 点「📱」按钮 → 弹出二维码
3. 点「下载二维码」→ 得到PNG图片
4. 把这个二维码发给包装印刷厂

### 不同批次对应不同二维码
```
批次A（李大哥）→ 二维码A → https://xxx/index.html?id=QJ20240601001
批次B（王大姐）→ 二维码B → https://xxx/index.html?id=QJ20240602001
批次C（张大伯）→ 二维码C → https://xxx/index.html?id=QJ20240603001
```

每个二维码扫描后显示对应批次的农户、日期、产地信息。

## 常见问题

**Q: 需要每个农户建一个仓库吗？**
A: 不需要。一个仓库管所有批次，只是二维码里的ID不同。

**Q: 修改信息后多久生效？**
A: GitHub Pages更新后1-2分钟生效。如果看到旧数据，清浏览器缓存或加`?t=1`参数。

**Q: 图片怎么更新？**
A: 在GitHub仓库直接替换同名图片文件即可。新批次的照片如果不同，可以命名为 `img_farmer_002.jpg` 并在admin中填写对应文件名。

**Q: admin.html 数据存在哪？**
A: 存在你浏览器本地（localStorage）。换电脑需要先用「导出JSON」备份，再在新电脑「导入JSON」。

**Q: 可以批量生成二维码吗？**
A: 在admin中逐个点📱按钮下载即可。如需批量，可以找我帮你写脚本。
